How this demo works

On-demand secure code scanning from the website

This public demo is designed to feel like a lightweight Checkmarx-style workflow while staying safe for a static website. The GitHub token remains server-side in Azure Functions; the browser only submits the repository URL and receives the report.

  • Accepts public GitHub repositories only
  • Reads selected source files through the GitHub API
  • Does not clone, build, run, or execute submitted code
  • Uses backend rule signals plus AI reasoning to produce OWASP findings
  • Returns findings, severity, evidence, and remediation guidance immediately
Demo scope: AI findings are indicators for review. A production SAST program should also include dependency scanning, data-flow analysis, secrets scanning, authentication testing, and manual validation.
