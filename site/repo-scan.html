How this demo works

Checkmarx-style positioning, lightweight demo boundaries

This demo is designed for website showcase and lead generation. It inspects public source files through the GitHub API, applies OWASP Top 10 source rules, and stores the request in the private owner dashboard.

  • Accepts public GitHub repositories only
  • Does not clone, build, run, or execute submitted code
  • Limits file count, file size, and total source characters scanned
  • Flags indicators for review rather than claiming confirmed exploitation
  • Uses local OWASP triage for prioritization and remediation summary; no external AI or paid model API is called
Demo scope: Findings are source-code indicators. A production SAST program should also include dependency scanning, data-flow analysis, secrets scanning, authentication testing, and manual validation.
The request and scan summary go to the private dashboard.
